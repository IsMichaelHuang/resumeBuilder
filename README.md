# Michael Huang - Resume System

A modular LaTeX resume system built with a **Python build pipeline** that assembles version-specific sections from shared snippets. Includes a **Monster Resume** (master document) and 4 tailored 1-page versions for different job types.

## How It Works

The build system uses a **snippet assembly** approach:

1. **Snippets** — Each project and skill category lives in its own `.tex` file under `sections/projects/` and `sections/skills/`.
2. **Config** — The `VERSIONS` dict in `build.py` maps each resume version to its selected project and skill snippets.
3. **Assembly** — At build time, Python reads the relevant snippets, wraps them in LaTeX section boilerplate, and writes the assembled `projects.tex` and `skills.tex` into each version directory.
4. **Compilation** — Python invokes `pdflatex` via `subprocess` and renames the output to `Michael_Huang_Resume_<Version>.pdf`.

This means projects and skills are **defined once** and **reused across versions** — no duplication.

## Monster Resume - Your Master Document

The **Monster Resume** at `monster/` contains ALL your experiences, projects, and skills. This is your **single source of truth**.

**Workflow:**
1. **Update monster first** - When you complete projects, learn skills, get new jobs, add to `monster/`
2. **Add snippets** - Create new `.tex` snippets in `sections/projects/` or `sections/skills/`
3. **Update config** - Add the snippet name to the relevant version(s) in `build.py`
4. **Build** - Run `python3 build.py all`
5. **Keep tailored to 1 page** - `versions/` are for applications (1 page only)
6. **Monster can be multi-page** - That's OK! It's your personal archive

See [monster/README.md](monster/README.md) for details.

## Quick Start

### 1. Choose the Right Resume Version

| Job Posting Mentions... | Use This Version |
|------------------------|------------------|
| AI, ML, LLM, PyTorch, Deep Learning, Neural Networks | `ai-ml` |
| React, Node, Full-Stack, MERN, Frontend, Backend | `fullstack` |
| Backend, APIs, Systems, Infrastructure, Linux, Microservices | `backend` |
| General Software Engineer, New Grad, SWE | `general-swe` |

### 2. Build the Resume

```bash
# Build a specific version
python3 build.py ai-ml
python3 build.py fullstack
python3 build.py backend
python3 build.py general-swe

# Build all 4 tailored versions at once
python3 build.py all

# Build the monster (master) resume
python3 build.py monster
```

**Output:** `Michael_Huang_Resume_<Version>.pdf` in the version's directory (e.g. `versions/ai-ml/Michael_Huang_Resume_AI_ML.pdf`)

## How to Edit

### Editing Shared Content

These files are used by **ALL versions**. Edit once, affects all:

- **Contact Info**: Edit `sections/heading.tex`
- **Education**: Edit `sections/education.tex`
- **Experience**: Edit `sections/experience.tex`
- **Hackathon**: Edit `sections/hackathon.tex`
- **Formatting**: Edit `preamble.tex`

After editing shared files, recompile all versions:
```bash
python3 build.py all
```

### Adding or Editing Projects

1. Create or edit a snippet in `sections/projects/` (e.g. `sections/projects/new_project.tex`)
2. Add the snippet name to the relevant version(s) in `build.py` under `VERSIONS`
3. Build: `python3 build.py all`

### Adding or Editing Skills

1. Create or edit a category snippet in `sections/skills/` (e.g. `sections/skills/cloud.tex`)
2. Add the snippet name to the relevant version(s) in `build.py` under `VERSIONS`
3. Build: `python3 build.py all`

### Editing Version-Specific Content

Only summaries remain version-specific:

- **Summary**: Edit `versions/[version]/summary.tex`

After editing:
```bash
python3 build.py [version]
```

## Technical Details

### Prerequisites
- **Python 3.10+** (for `build.py`)
- **MacTeX** (installed via `brew install --cask mactex`)
- **pdflatex** command available in PATH

### File Paths Explained
Each version's `main.tex` uses relative paths:
```latex
\input{../../preamble}            % Goes up 2 levels to /resume/preamble.tex
\input{summary}                   % Loads summary.tex from same directory
\input{../../sections/heading}    % Goes up 2 levels to /resume/sections/heading.tex
\input{../../sections/experience} % Shared experience across all versions
\input{projects}                  % Generated by build.py into this directory
\input{skills}                    % Generated by build.py into this directory
```

## Troubleshooting

### "pdflatex: command not found"
```bash
# Add to PATH
export PATH="/Library/TeX/texbin:$PATH"

# Or restart your terminal
```

### "File not found: ../../preamble.tex"
- Make sure you're in the correct directory
- Should be in `versions/[version]/` when compiling manually
- Check that `preamble.tex` exists in the root `/resume/` directory

### "LaTeX Error: File '*.sty' not found"
- Missing LaTeX package
- Install with: `sudo tlmgr install [package-name]`
- Or use MacTeX which includes all packages

### Resume exceeds one page
1. Reduce margins (edit `preamble.tex`)
2. Remove a project or bullet point
3. Tighten vertical spacing
4. **Don't** reduce font size below 11pt (ATS may have issues)

### Changes not showing in PDF
- Make sure you saved the `.tex` file
- Recompile: `python3 build.py [version]`
- Sometimes need to delete auxiliary files: `rm versions/[version]/*.aux versions/[version]/*.log`
